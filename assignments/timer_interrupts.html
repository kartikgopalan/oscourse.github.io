Kernel Module and Timers

Objective

You will learn how to develop and use kernel modules. You will also learn how the Linux OS uses High-Resolution Timers in kernel space.

Problem Description: Write a kernel module to generate 1000 interrupts per second using hrtimer subsystem.

Create a kernel module that intializes the hrtimer and fires an interrupt every 1 ms. Measure if the timer interrupt fired at the expected time in nanoseconds.

Below we describe all the resources you will need to learn and/or use to completethis assignment. Start early and ask us for help if you get stuck.

Instructions to access your virtual machine remotely

TASK 1

Learn how to write and execute a Linux kernel module.

Kernel modules allow the code to be dynamically added to the kernel, dynamically. Try compiling and inserting the kernel modules hello.c and hellon.c from kernel module slides.

  1. Save the Makefile, hello.c, and hellon,c in your home directory. The files hello.c has initialization and cleanup functions that are invoked upon loading and unloading the kernel module. The second file hellon.c is similar, except that it takes command-line arguments with insmod command to print a custom message.
  2. Compile the kernel module: 
    make
  3. Load the kernel module: 
    sudo insmod hello.ko

    If the module was successfully inserted, you will see "mymodule: Hello World!" message in kernel log. Use commands "dmesg" or "cat /var/log/kern.log" to see the kernel log messages.

  4. Unload the kernel module: 
    sudo rmmod hello

    Upon removing the kernel module, use the "dmesg" command to check if kernel log prints the message "mymodule: Goodbye, cruel world!!"

  5. For hellon,c, module insertion command is sudo insmod hellon.ko whom=class howmany=10 (or any other argument values you want to give). Removal is the same as for hello.c.

  6. Submit a kernel module practice_module.c that takes loop_count as kernel module parameter, and prints the message "Hello world!" loop_count times on inserting the module.

TASK 2

Understand how to initialize and start the timers in the Linux kernel using hrtimers.

  1. hrtimers are software timers in Linux kernel used to schedule timer interrupts. Hrtimers use ktime to get the current time and to set the expiry time at which the timer interrupt fires.
  2. hrtimer structure has a number of fields including expiry time and a callback function on time expiration struct hrtimer.
  3. Initialize hrtimer and register a callback function using hrtimer_init().
  4. The timer interval or expiry time at which the timer fires an interrupt is passed as a kernel module parameter, timer_interval.
  5. Get the current time and set the expiry time using the ktime related functions to generate timer interrupts at 1000 interrupts per second. Start the timer using hrtimer_start().
  6. Callback function: A callback function executes when the timer expires. The callback function also reset the timer to fire 1000 interrupts in the next second using hrtimer_forward().
  7. Finally, cancel the timer using hrtimer_cancel() when the timer is no longer needed.

TASK 3

Compare the accuracy in user space and kernel space.

  1. Add another kernel module parameter, message_count, to define the number of print messages.
  2. Record the current time, expected expiry time, actual expire time (when the interrupt fires) and difference in the actual and expected expiry time
  3. Print the recorded values when the kernel module is removed.
  4. Compare the values measured in the kernel module to the values observed in the user space using the program userspace_timer.c.
  5. Report the observations.
